# Spoon River Anthology Mention Network

This repository hosts the interactive web infrastructure for exploring the interpoetic network of Edgar Lee Masters' *Spoon River Anthology*. It provides a dynamic visualization of character mentions across the poems, allowing users to investigate narrative connections, central characters, and network patterns.

## Project Overview

In her introduction to a 1962 edition, May Swenson describes the *Spoon River Anthology* as “fascinating if nothing else than to untangle the lurid web of small-town scandals provocatively placed before us.” Her words perfectly convey the reading experience of the work, and thanks to this project, that very **web of scandals** is now available to explore interactively, together with its full dataset, making the narrative structure tangible and accessible.

This project visualizes the mention interactions among the characters of the Anthology, aiming to provide an open and interactive resource for scholars, students, and literature enthusiasts.

## Access the Dataset

The structured dataset used in this project is hosted in the [`database` branch`](https://github.com/MMoscato00/SpoonRiverAnthologyDH/tree/database), containing:

- **Edges**: character mentions between poems, with information about type and edition  
- **Nodes**: poems, with unique identifiers and edition information  

This allows anyone to replicate analyses, explore the dataset in their own environment, or create alternative visualizations.

## How to View

To explore the interactive network:

1. **Local Viewing:**  
   Download or clone the repository, make sure all files (`index.html`, `mygraph.html`, CSS, JS if any) are in the same folder, and open `index.html` in your browser.

2. **GitHub Pages:**  
   The project may also be available via GitHub Pages at [https://mmoscato00.github.io/SpoonRiverAnthologyDH/](https://mmoscato00.github.io/SpoonRiverAnthologyDH/), if deployed.  

> **Note:** Make sure all HTML, CSS, and JS files are together in the same folder when opening locally for the visualization to work correctly.
